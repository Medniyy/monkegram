# MonkeGram — dApp Store submission form (copy-paste reference)

Maps **1:1** to the Publisher Portal "Let's add your dApp details!" form
(https://publish.solanamobile.com). Fill top-to-bottom. Values match the
live, published v1.0.2 listing — for an update, keep them identical unless a
field changed. Companion docs: `LISTING.md` (source of truth for copy),
`RELEASE_GUIDE.md` (APK build + on-chain release).

> **Is the listing form even needed for this update?** This "dApp details" form
> is the **listing metadata** and is normally already filled / pre-populated for
> an existing app. A *new version* only requires the **APK + "What's new"** step
> (see §2 below). Only re-touch the fields below if an asset or copy actually
> changed. **Web-only changes (e.g. the MonkeAPI routing) need NO store
> submission at all** — they go live via gh-pages and the WebView re-fetches.

## 2. New-version (APK) step — separate from the details form above

Only required when **native** (`monkegram-mobile/`) code changed. Do this before
or after the details form; they're independent.

1. **Bump version** in `monkegram-mobile/app.json` — `expo.version` +
   `expo.android.versionCode` **+1** (current = `1.0.2` / code `3`, so next =
   `1.0.3` / code `4`). The store rejects an upload whose versionCode isn't higher.
2. Build + sign the APK per `RELEASE_GUIDE.md` §2 (space-free `C:\dev\mg`,
   `prebuild --clean`, `assembleRelease arm64-v8a`, **never `gradlew clean`**;
   signed with the irreplaceable `monkegram-dappstore.keystore`).
3. On the Portal: open **MonkeGram → New Version / Release**, upload the **APK**
   (not AAB), enter **What's new**, approve the on-chain release (~0.2 SOL).
4. Review ~3–5 days from `publishersupport@dappstore.solanamobile.com`.

**Reviewer notes** (paste): MonkeGram requires a compatible Solana Mobile Wallet
Adapter wallet to enter. Connecting is **sign-in only (MWA / Sign-In With
Solana) — no token transfer**, no fund-bearing actions. After entry, choose any
SMB Gen2 or Gen3 by number; grant camera + mic; record a short clip; verify
**POST TO X** (primary) and **SAVE TO DEVICE** (secondary). Clips are created
locally and never uploaded to a MonkeGram backend.

**What's new — template** (fill per release):

> v1.0.3 — <one-line summary of what changed in this build>

## 3. Submission must-haves (don't lose these)

- **Publisher wallet** — holds the Publisher + App NFTs; every future release
  must originate from this same wallet. Seed backed up offline.
- **Release keystore** — `monkegram-mobile/credentials/monkegram-dappstore.keystore`
  (alias `monkegram`, store+key pwd `MonkeGramDappStore2026!`). Lose it = can
  never update this listing again, only republish as a new app. Duplicate at
  `C:\dev\mg\credentials\`.
